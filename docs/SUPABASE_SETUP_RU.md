# Пошаговая настройка Supabase (бесплатно) для «Монеты»

Эта инструкция для **новичков**: что нажать, куда вставить ключи, сколько это стоит.

## Сколько стоит

- **Supabase**: есть **бесплатный** тариф (ограничения по месту в БД и трафику — для старта и тестов обычно хватает).
- **Хостинг фронта** (Vercel / Netlify / Cloudflare Pages): тоже есть **бесплатные** планы.
- **Ваш личный бюджет до 3000 ₽/мес.** на старте можно **не тратить**, если пользоваться только free tier и не подключать платные SMS и т.п.

Если позже понадобится больше мощности или домен — тогда закладывайте расходы отдельно.

---

## Шаг 1. Регистрация в Supabase

1. Откройте в браузере: **https://supabase.com**
2. Нажмите **Start your project** / **Sign up** (можно через GitHub или email).
3. Подтвердите почту, если попросят.

---

## Шаг 2. Создать проект

1. В панели нажмите **New project**.
2. Укажите **имя проекта** (любое, например `moneta-prod`).
3. **Database password** — придумайте **сложный пароль** и **сохраните его** в менеджере паролей (он нужен для прямого доступа к БД; для приложения мы используем другие ключи).
4. **Region** — ближайший к вам (например `Frankfurt` для Европы).
5. Нажмите **Create new project** и подождите 1–2 минуты, пока проект создаётся.

---

## Шаг 3. Включить таблицы и безопасность (SQL)

1. В левом меню выберите **SQL Editor**.
2. Нажмите **New query**.
3. Откройте в репозитории файл **`supabase/migrations/001_initial.sql`**, скопируйте **весь** текст и вставьте в редактор.
4. Нажмите **Run** (или Ctrl+Enter).
5. Внизу должно быть сообщение об успехе. Если есть ошибка про `execute function` / `execute procedure` — ваша версия PostgreSQL может отличаться. Попробуйте в последней строке триггера заменить:
   - `execute function public.handle_new_user()`  
   на  
   - `execute procedure public.handle_new_user()`  
   и снова выполнить скрипт.

Этот скрипт создаёт таблицы `profiles`, `transactions`, `budget_limits`, включает **RLS** (пользователь видит только свои строки) и триггер, который при регистрации создаёт строку в `profiles`.

---

## Шаг 4. Взять ключи API для приложения

1. Слева откройте **Project Settings** (шестерёнка).
2. Раздел **API**.
3. Найдите:
   - **Project URL** — что-то вроде `https://xxxx.supabase.co`
   - **anon public** ключ — длинная строка, начинается с `eyJ...`

**Важно:** ключ **anon** можно вставлять **только во фронтенд**, если включён RLS (у нас включён). Секретный **service_role** в браузер **не** кладите.

---

## Шаг 5. Подключить ключи к проекту на компьютере

1. В папке **`web`** создайте файл **`.env.local`** (его нет в git — это нормально).
2. Скопируйте шаблон из **`web/.env.example`** и подставьте свои значения:

```env
VITE_SUPABASE_URL=https://ВАШ_ПРОЕКТ.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
```

3. Сохраните файл.
4. Остановите `npm run dev`, если он был запущен, и запустите снова:

```bash
cd web
npm run dev
```

Vite подхватывает переменные только при старте.

---

## Шаг 6. Подтверждение email (рекомендация для разработки)

По умолчанию Supabase может **требовать подтверждение почты** после регистрации.

- **Для тестов** можно ослабить: **Authentication → Providers → Email → Confirm email** — временно выключить, чтобы сразу входить после регистрации.
- **Для продакшена** лучше оставить подтверждение и настроить письма.

---

## Шаг 7. Проверка в приложении

1. Откройте сайт локально (адрес из терминала).
2. В шапке нажмите **Войти** (или аналог на английском).
3. Зарегистрируйте аккаунт (email + пароль не короче 6 символов).
4. После входа в шапке должны отображаться почта и статус **«Данные в облаке»**.
5. Добавьте операцию — через ~1 секунду она должна сохраниться в таблице `transactions` (проверка: **Table Editor → transactions** в Supabase).

Если облако **не** настроено (нет `.env.local`), приложение работает **как раньше** — только `localStorage` в браузере.

---

## Шаг 8. Деплой на хостинг (когда будете готовы)

1. Залейте код в GitHub/GitLab.
2. Подключите репозиторий к **Vercel** или **Netlify**.
3. Укажите **Root directory**: `web`, команда сборки: `npm run build`, папка вывода: `dist`.
4. В настройках проекта добавьте **те же** переменные `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` (Environment Variables).
5. Пересоберите деплой.

---

## Что дальше по плану

- Оплата подписок (Stripe / ЮKassa) — отдельный этап: webhook → обновление поля `plan` в `profiles`.
- Админ-панель — отдельные роли в БД и защищённые маршруты.
- 2FA по SMS — почти всегда платно; TOTP (приложение-аутентификатор) можно добавить дешевле.

Если что-то из шагов «не завелось» — смотрите консоль браузера (F12) и логи в Supabase **Logs**.
